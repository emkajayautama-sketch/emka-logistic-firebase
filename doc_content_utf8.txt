DOKUMEN SPESIFIKASI STRUKTUR MENU APLIKASIINTEGRASI JASA PBM & JPTSistem Informasi Logistik Pelabuhan Terpadu (SILOP)Kategori Usaha: Umum (Non-Kontainer, Curah Kering, Curah Cair, & General Cargo)I. PENDAHULUAN DAN TUJUAN SISTEMSistem Informasi Logistik Pelabuhan Terpadu (SILOP) dirancang khusus untuk mengakomodasi kebutuhan operasional perusahaan Jasa Perusahaan Bongkar Muat (PBM) dan Jasa Pengurusan Transportasi (JPT). Sistem ini bersifat umum, mengelola tidak hanya komoditas peti kemas (kontainer), melainkan komoditas non-kontainer seperti barang curah kering (pasir besi, batu, semen), curah cair (BBM, aspal), dan general cargo (besi beton, tiang pancang).Tujuan utama sistem ini adalah mengintegrasikan sirkulasi dokumen administrasi, pencatatan eksekusi fisik dilapangan secara real-time melaui sistem Tally Digital, otomatisasi rekapitulasi laporan regulasi bulanan/tahunan untuk KSOP, serta penyediaan jalur akses cepat (shortcut link) ke portal eksternal seperti Inaportnet tanpa terjadi kendala double-input.II. WORKFLOW ALUR KERJA SISTEM (END-TO-END)Alur sirkulasi data dan aktivitas pengguna di dalam sistem diatur mengikuti kronologi pergerakan kapal dan pengurusan barang sebagai berikut:Tahap Pra-Kedatangan: Klien memesan jasa logistik. Staf Admin JPT menginput data kapal dan muatan di Menu 2. Sistem secara otomatis menyediakan format teks ringkas untuk memudahkan admin melakukan copy-paste data ke portal resmi Inaportnet melalui menu jalan pintas (Menu 5).Tahap Perencanaan Lapangan: Staf Admin PBM menyusun Rencana Bongkar Muat (RBM) serta menetapkan alokasi kelompok kerja (Geng TKBM) dan alat berat pendukung di Menu 3.1.Tahap Eksekusi Lapangan: Kapal sandar. Petugas Tally Lapangan melakukan pencatatan real-time hasil bongkar/muat menggunakan tablet/handphone pada Menu 3.2. Setiap truk pengangkut komoditas dibekali Surat Jalan Digital ber-QR Code.Tahap Validasi Gerbang: Truk melewati jembatan timbang menuju pintu keluar pelabuhan. Petugas gate memindai QR Code menggunakan Menu 4. Sistem secara otomatis menarik angka berat bersih barang (netto) dari jembatan timbang dan menguncinya ke database.Tahap Finalisasi & Pelaporan: Selesai operasional, sistem secara otomatis mengunci angka aktual ke dalam dokumen Outturn Report, menerbitkan draf Invoice Tagihan di Menu 7, serta menyusun berkas Laporan Bulanan KSOP di Menu 6.IV. CATATAN TEKNIS DAN PENGEMBANGAN (DEVELOPMENT NOTES)1. Desain UI Khusus Lapangan (Mobile Responsive)Modul Tally Lapangan (Menu 3.2) dan Scanner Portal (Menu 4) wajib dikembangkan dengan prinsip mobile-first. Tampilan antarmuka harus menggunakan tombol berukuran besar, skema warna kontras tinggi agar tetap terlihat jelas di bawah terik matahari dermaga, serta pemrosesan data yang ringan.Rekomendasi Keamanan Sistem (Role Permissions):Hak akses sistem wajib dibatasi ketat menggunakan User Roles (Menu 8.1). Petugas Tally Lapangan hanya diberikan hak akses menulis pada modul input harian dan dilarang keras melihat data keuangan (Menu 7) ataupun menu konfigurasi master.2. Integrasi Tautan Inaportnet (Webview Shortcuts)Mengingat kebijakan keamanan siber server kementerian menutup akses API langsung dari sistem swasta, integrasi dilakukan menggunakan tautan langsung (Deep Linking) yang tertanam di dalam aplikasi. Untuk mempercepat proses administrasi, sistem internal wajib menyediakan fitur Teks Generator Otomatis yang mengompilasi manifest menjadi draf siap salin (copy), sehingga meminimalkan kesalahan pengetikan ulang (human error) oleh staf administrasi.
